# 条件設定・実行タブ: スキャン処理性能改善 要件定義

Status: target
Updated: 2026-04-01

## 1. 目的
- Parquet スキャン開始から観測所一覧表示までの待ち時間を短縮する。
- 大量ファイル・大容量データ環境でも、UI操作の応答性を維持する。

## 2. スコープ
- 対象:
  - スキャン処理の2段階化（軽量スキャン / 詳細読み込み）
  - 観測所一覧表示に必要な列だけの先行読込
  - スキャン結果キャッシュ戦略の拡張
- 非対象:
  - 描画ロジックの変更
  - 実行前検証・バッチ実行の判定仕様変更

## 3. 機能要件

### R-SPR-01 2段階スキャン
- スキャンは以下の2段階で処理する。
  - Phase 1（軽量）: 観測所一覧表示のための最小情報のみ読込
  - Phase 2（詳細）: 実行前検証・プレビュー・バッチ実行時に必要な詳細データ読込
- `スキャン` ボタン押下時は Phase 1 のみ実行し、結果をUIへ反映する。

### R-SPR-02 観測所一覧の列限定読込
- Phase 1 では次の列に限定して Parquet を読込む。
  - `source`
  - `station_key`
  - `station_name`
- 基準日候補の先行算出が必要な場合に限り、`observed_at` の追加読込を許可する（初期は無効）。

### R-SPR-03 詳細読込の遅延実行
- Phase 2 は以下のタイミングでのみ実行する。
  - 実行前検証開始時
  - プレビュー生成時
  - バッチ実行開始時
- Phase 2 実行済みで入力ディレクトリ内容が不変なら、再読込を避ける。

### R-SPR-04 キャッシュ拡張
- 既存のディレクトリフィンガープリント（`path + size + mtime_ns`）を維持しつつ、
  - Phase 1 用キャッシュ
  - Phase 2 用キャッシュ
  を分離して保持する。
- ディレクトリ内容が変化したら、該当キャッシュを無効化する。

### R-SPR-05 UI状態表示
- スキャン完了メッセージは「軽量スキャン完了」であることが判別できる文言にする。
- Phase 2 初回実行時は、必要に応じてログに「詳細読込開始/完了」を出す。

## 4. 受け入れ条件
1. `スキャン` 押下後、観測所一覧が現行より短時間で表示される。
2. `スキャン` 直後は重い詳細読込が走らない。
3. 実行前検証・プレビュー・バッチ実行は従来どおり成立する（必要時に詳細読込される）。
4. 同一ディレクトリ再スキャン時にキャッシュが効き、不要な再読込が抑制される。
5. ディレクトリ内容更新時はキャッシュが無効化され、最新内容が反映される。

## 5. テスト観点（最小）
- 正常:
  - 軽量スキャンで観測所一覧表示
  - 詳細読込トリガー（precheck/preview/batch）
  - 2種類キャッシュのヒット/ミス
- 回帰:
  - 不正Parquet検出（invalid_files/warnings）
  - 既存の実行前検証結果とバッチ出力結果
