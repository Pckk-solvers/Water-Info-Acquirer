# Hydrology Graphs Platform 改善バックログ

Status: target
Updated: 2026-04-01

## 1. 目的
- 次フェーズで改善したい項目を蓄積し、仕様検討の起点にする。
- 実装前に「要求」「想定仕様」「影響範囲」「論点」を整理する。

## 2. 改善項目（今回整理）

### ✅ B-01 条件設定・実行タブの結果表示統合（完了）
- 要求:
  - `実行前検証結果` と `実行結果` を統合表示したい。
- 想定仕様:
  - 1つの結果テーブルで、状態を `precheck_ng / ready / success / failed / skipped` のように管理する。
  - 実行前検証時は「実行可否」を表示し、実行後は同じ行の状態を更新する。
- 期待効果:
  - 判定結果と実行結果の対応関係が追いやすくなる。
- 主要影響:
  - UI（Treeview定義）
  - ViewModel（状態モデル）
  - Service結果のマッピング

### ✅ B-02 基準日入力をプルダウン化（自由記述廃止）（完了）
- 要求:
  - `YYYY-MM-DD` 自由記述ではなく、候補から選んで追加したい。
- 想定仕様:
  - 候補日コンボボックス（Parquetスキャン結果由来）を選択し、「追加」で選択リストに積む。
  - 追加済み基準日は重複不可、削除可能。
- 期待効果:
  - 入力ミス防止、検証エラー削減。
- 主要影響:
  - UI（Text -> Combo + List管理）
  - ViewModel（base_datesの管理）
  - 実行前検証入力生成

### ✅ B-03 基準日リストのCSVエクスポート/インポート（完了）
- 要求:
  - 基準日リストを外部保存・再利用したい。
- 想定仕様:
  - 形式: CSV（単一列 `base_date`）
  - UTF-8（BOMあり/なし許容）
  - インポート時に不正日付・重複を除外して警告表示
- 期待効果:
  - 条件再利用、運用効率化。
- 主要影響:
  - UI（保存/読込ボタン）
  - IOヘルパー（CSV read/write）

### ✅ B-04 条件設定・実行タブを2ペインに再構成（完了）
- 要求:
  - 左側に設定/実行、右側に結果/ログを配置したい。
- 想定仕様:
  - 左ペイン: 入力群 + 実行ボタン群
  - 右ペイン: 統合結果テーブル + ログ
- 期待効果:
  - 実行導線が明確になり、画面視線移動が減る。
- 主要影響:
  - UIレイアウト（Frame構成）
  - ウィジェットの再配置

### ✅ B-05 イベント窓（3日/5日）を同時出力可能にする（完了）
- 要求:
  - 現状の「3日 or 5日」排他ではなく、両方を出力対象にしたい。
- 想定仕様:
  - イベント窓選択を単一ラジオから複数選択（チェック）へ変更。
  - イベント系グラフは選択窓ごとに独立ターゲットを生成する（例: 3日と5日の2件）。
- 期待効果:
  - スタイルを分けた9種運用と整合する。
- 主要影響:
  - UI（窓選択）
  - Service入力DTO（`event_window_days: int` -> `event_window_days_list: list[int]` など）
  - Precheck/Batchのターゲット展開

## 4. 新規改善項目（2026-04-01 追加）

### ✅ B-06 基準日候補を「選択観測所連動」にする（完了）
- 要求:
  - 基準日候補は全体日付ではなく、選択中観測所で有効な日付のみを表示したい。
- 想定仕様:
  - 観測所選択変更イベントで基準日候補を再計算。
  - 候補は「選択観測所の少なくとも1つに実データがある日」を対象にする。
  - 追加済み基準日のうち候補外になったものは保持するが、非推奨表示（要検討）。
- 主要影響:
  - UI（観測所選択イベント処理）
  - ViewModel（候補日算出）
  - Catalogアクセス（観測所別日付インデックス）

### ✅ B-07 観測所選択UIの操作性/性能改善（完了）
- 要求:
  - 観測所選択が使いづらく、大量選択時に操作ラグが出る問題を改善したい。
- 想定仕様（候補）:
  - チェック式の観測所選択UIへ変更
  - `全選択/全解除` を追加
  - 行クリックだけでは意図せず解除されにくい操作にする
  - UI更新のバッチ化（逐次再描画を抑制）
- 主要影響:
  - UI（Listbox操作）
  - ViewModel（フィルタ状態）
  - パフォーマンス測定（スキャン済み大量件数）

### ✅ B-08 観測所表示名の日本語化（完了）
- 要求:
  - 観測所選択の `source` を内部値ではなく業務向け表示名で見せたい。
- 想定仕様:
  - `jma` -> `気象庁`
  - `water_info` -> `水文水質DB`
  - 表示形式: `<source_label>:<station_key> (<station_name>)`
- 主要影響:
  - UI（観測所リスト表示）
  - ViewModel（表示文字列生成）

### ✅ B-09 条件設定・実行タブの左右ペイン比率安定化（完了）
- 要求:
  - 初期表示時に左右ペインを 1:1 に近い見た目で固定したい。
- 想定仕様:
  - 左右ペインは同ウェイトを基本とする。
  - 右ペインの結果テーブル列幅を見直し、過大要求幅を抑える。
- 主要影響:
  - UIレイアウト（grid weight）
  - 結果テーブル列定義（初期幅/伸縮）
