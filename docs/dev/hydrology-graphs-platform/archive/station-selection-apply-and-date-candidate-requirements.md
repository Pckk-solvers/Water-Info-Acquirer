# 条件設定・実行タブ: 観測所選択反映と基準日候補遅延計算 要件定義

Status: target
Updated: 2026-04-01

## 1. 目的
- 観測所チェック操作ごとに基準日候補探索が走ることで発生する体感遅延を減らす。
- ユーザーが任意のタイミングでまとめて反映できる操作導線に変更する。

## 2. スコープ
- 対象:
  - 観測所選択後の基準日候補探索トリガー制御
  - 「チェック反映」ボタン追加
  - 基準日設定エリア上段ボタン配置変更
- 非対象:
  - 基準日候補算出ルール（和集合）の変更
  - 実行前検証・バッチ実行ロジックの仕様変更

## 3. 機能要件

### R-CSA-01 基準日候補探索の遅延実行
- 観測所チェックON/OFF時には基準日候補探索を実行しない。
- 観測所チェック状態はUI上に即時反映するが、候補探索は保留する。
- 候補探索は「チェック反映」ボタン押下時にのみ実行する。

### R-CSA-02 チェック反映ボタン
- 基準日設定エリアに `チェック反映` ボタンを追加する。
- 押下時に、現在のチェック済み観測所集合を使って基準日候補（和集合）を再計算する。
- 押下時に候補が空の場合は、候補プルダウンを空表示とする（既存仕様に準拠）。

### R-CSA-03 ボタン配置
- 基準日設定エリア1行目の左端に `チェック反映` を配置する。
- 同じ1行目で `追加` と `削除` を右寄せで配置する。
- `全削除`、`CSV読込`、`CSV保存` の並びは現行仕様を維持する。

### R-CSA-04 反映未実行の扱い
- チェック変更後に反映していない場合、直前に反映済みの候補を保持する。
- 反映前に実行前検証を開始した場合は、最後に反映済みの候補・基準日で処理する。

## 4. 受け入れ条件
1. 観測所チェックON/OFF操作だけでは基準日候補探索が走らない。
2. `チェック反映` 押下でのみ候補探索が実行される。
3. 基準日設定1行目の左に `チェック反映`、右に `追加`/`削除` が表示される。
4. 大量観測所で連続チェック操作しても体感遅延が改善している。
5. 候補算出ルール（選択観測所の和集合）は従来どおり維持される。

## 5. テスト観点（最小）
- 正常:
  - チェック変更のみ（候補未更新）
  - チェック反映押下（候補更新）
  - 追加/削除ボタンの配置と動作
- 回帰:
  - 全選択/全解除後の反映
  - 基準日CSV読込/保存
  - 実行前検証とバッチ実行の既存挙動
