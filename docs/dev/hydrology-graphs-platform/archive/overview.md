# Hydrology Graphs Platform Overview

## 1. 目的
- Parquet を入力にして、水文可視化を再生成できる独立機能（将来的には独立パッケージ）を作る。
- ユーザーが GUI 上で見た目を調整し、そのスタイルで複数グラフを一括出力できるようにする。
- スタイルは JSON で保存し、再利用できるようにする。

## 2. 作成するグラフ一覧（全6種）
### 2.1 1回の降雨・出水イベントを対象としたグラフ（時間変化）
1. ハイエトグラフ（降雨柱状図）
   - 対象: 時間雨量・累加雨量
   - 仕様: 上から下へ垂れ下がる棒グラフ（Y軸反転）
2. ハイドログラフ（流量波形）
   - 対象: 流量
   - 仕様: 折れ線グラフ + 基準線（例: 計画高水流量）
3. 水位波形（水位ハイドログラフ）
   - 対象: 水位
   - 仕様: 折れ線グラフ + 防災基準線（例: 計画高水位(HWL), 避難判断水位）

### 2.2 過去数十年間の歴史・安全度を対象としたグラフ（経年変化）
4. 年最大雨量の経年変化図
   - 対象: 各年の最大雨量
   - 仕様: 棒グラフ（X軸=年） + 水平線（例: 計画降雨量, 平均値）
5. 年最大流量の経年変化図
   - 対象: 各年の最大流量
   - 仕様: 棒グラフ（X軸=年） + 水平線（例: 整備計画流量）
6. 年最高水位の経年変化図
   - 対象: 各年の最高水位
   - 仕様: 棒グラフ（X軸=年） + 水平線（例: 計画高水位(HWL)）

## 3. メトリクス対応方針
- ハイエトグラフ: 雨量（rainfall）
- ハイドログラフ（流量波形）: 流量（discharge）
- 水位波形: 水位（water_level）
- 年最大雨量: 雨量（rainfall）
- 年最大流量: 流量（discharge）
- 年最高水位: 水位（water_level）

## 4. データ前提（粗案）
- 入力は共通時系列スキーマの Parquet を前提とする。
- Parquet 仕様の詳細は `../current/parquet-contract.md` を単一の参照元として扱う。
- イベント系グラフ（2.1 の3種）は 1時間間隔データが必須。
- イベント系グラフの対象期間は「基準日を含む 3日または5日」を基本とする。
- 経年系グラフ（2.2 の3種）は長期間データを前提にする（複数年〜数十年）。

## 5. 操作イメージ（GUI）
本機能は同一画面内の 2 タブ構成とする。

1. 条件設定・実行タブ
- Parquet ディレクトリを指定し、作成可能な観測所・メトリクス・期間を一覧化する。
- グラフ種別（全6種）、観測所（複数）、基準日（複数）、3日/5日（イベント系）を選択する。
- 実行前検証（データ不足、契約違反、年数不足）を確認する。
- バッチ生成を実行し、成功/失敗/スキップ理由と出力先を確認する。
- ログを確認する。

2. スタイル調整タブ
- プレビュー対象を選択し、スタイル設定（線色、太さ、フォント、凡例、軸、余白など）を調整する。
- プレビュー更新で見た目を確認する。
- 本タブでは実行機能を持たない（実行は条件設定・実行タブのみ）。

## 6. ランチャー統合方針（Overview）
- 本機能は独立ツールとして、既存ランチャー（`src/water_info_acquirer/launcher.py`）から起動可能。
- 起動導線は既存アプリと同じ方式（アプリ登録経由）で実装済み。

## 7. スタイル管理（JSON）
- スタイル定義は JSON ファイルとして保存する。
- GUI から「保存」「読み込み」「初期化」を行えるようにする。
- プリセット（例: 業務標準、報告書向け、プレゼン向け）を持てる構造にする。
- 詳細契約は `style-contract.md` を参照する。

## 8. 非機能の方向性
- バッチ出力時に進捗・成功/失敗・スキップ理由を明示する。
- 不足データがある対象は事前チェックで弾けるようにする。
- 同一条件で再実行しても再現性のある出力を目指す。

## 9. 今後詰める論点
- 3日/5日窓の厳密な定義（基準日の含め方、時刻境界、欠損許容）。
- 年最大グラフで必要とする最小年数・欠測許容率。
- スタイル JSON スキーマ（必須キー、バージョン管理、互換性）。
- 出力命名規則と上書きポリシー。
- 将来の独立パッケージ化時の境界（UI / ドメイン / 出力の分離）。
