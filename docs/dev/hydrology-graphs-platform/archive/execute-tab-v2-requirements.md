# 条件設定・実行タブ v2 要件定義

Status: target
Updated: 2026-04-01 (rev.4)

## 0. 実装ステータス
- ✅ 実装済み: R-EXE-01〜R-EXE-07
- ✅ 実装済み: R-EXE-01〜R-EXE-11

## 1. 目的
- 条件設定・実行タブの操作性を改善し、イベント窓の3日/5日同時出力を可能にする。
- 本書は実装着手可能な粒度で要件・受け入れ条件・完了条件を定義する。

## 2. スコープ
- 対象:
  - 結果表示の統合（実行前検証結果 + 実行結果）
  - 基準日入力UIの変更（自由記述廃止）
  - 基準日リストのCSV Import/Export
  - 条件設定・実行タブの2ペイン再構成
  - イベント窓の複数選択（3日/5日同時）
- 非対象:
  - スタイル調整タブの仕様変更
  - 描画アルゴリズム自体の変更
  - Parquet契約/threshold契約の変更

## 3. 機能要件

### R-EXE-01 結果表示の統合
- 1つの結果テーブルで以下を一元表示する。
  - 実行前検証の可否
  - バッチ実行結果
- 行キーは以下の複合キーで一意とする。
  - `source + station_key + graph_type + base_date + event_window_days`
- ステータス値:
  - `precheck_ng`, `ready`, `success`, `failed`, `skipped`

### R-EXE-02 基準日入力UI
- `YYYY-MM-DD` 自由記述入力を廃止する。
- 基準日候補は Parquet スキャン結果から取得した日付一覧を使う。
- UIは以下を持つ。
  - 候補日プルダウン
  - `追加` ボタン
  - 追加済み基準日リスト
  - `削除` ボタン（選択項目）
  - `全削除` ボタン
- 重複追加は不可。

### R-EXE-03 基準日CSV入出力
- 追加済み基準日リストをCSVでエクスポートできる。
- CSVをインポートして基準日リストへ反映できる。
- CSV仕様:
  - ヘッダ必須: `base_date`
  - 文字コード: UTF-8（BOMあり/なし許容）
  - 日付形式: `YYYY-MM-DD`
- インポート時の不正値/重複はスキップし、件数を警告表示する。

### R-EXE-04 2ペインレイアウト
- 条件設定・実行タブを左右2ペインにする。
  - 左ペイン: 設定入力 + 実行操作
  - 右ペイン: 統合結果テーブル + ログ

### R-EXE-05 イベント窓複数選択
- イベント窓設定はチェックボックス化する。
  - `3日`
  - `5日`
- どちらも未チェックの場合:
  - 実行前検証・バッチ実行ともエラーで中断する。
- 年最大系グラフではイベント窓設定を無視する（適用しない）。
- イベント系グラフは選択された窓ごとに独立ターゲットを生成する。

### R-EXE-06 DTO/モデル統一
- 単一値 `event_window_days` は廃止し、以下へ統一する。
  - `event_window_days_list: list[int]`
- 許容値は `3` と `5` のみ。

### R-EXE-07 出力パス
- イベント系の出力は窓情報をディレクトリに含める。
  - `<output_dir>/<station_key>/<graph_type>/<base_date>/<3day|5day>/graph.png`
- 年最大系は従来どおり窓ディレクトリなし。
  - `<output_dir>/<station_key>/<graph_type>/annual/graph.png`

### R-EXE-08 基準日候補の観測所連動
- 基準日候補は「現在選択されている観測所」で有効な日付のみ表示する。
- 観測所選択が変わったら、基準日候補を再計算する。
- 候補算出は以下の既定とする。
  - 既定: 選択観測所の**和集合**（どれか1観測所に存在する日付を候補化）
  - 将来拡張: 積集合モードは別要件で扱う

### R-EXE-09 観測所選択UI改善
- 観測所選択UIをチェック式に変更する。
- 一括操作を追加する。
  - `全選択`
  - `全解除`
- 意図しない選択解除が起きにくい操作設計にする。
- 大量件数時の体感遅延を抑える（再描画回数を抑制）。

### R-EXE-10 観測所表示名の日本語化
- 観測所選択リストの `source` 表示は内部値ではなく日本語表示名を使う。
  - `jma` -> `気象庁`
  - `water_info` -> `水文水質DB`
- 観測所の表示形式は以下とする。
  - `<source_label>:<station_key> (<station_name>)`
  - 観測所名が空の場合は `(<station_name>)` を省略する。
- 内部ロジック（識別キー、リクエスト、保存）は従来どおり `source` の内部値を使用する。

### R-EXE-11 左右ペイン比率の安定化（1:1）
- 条件設定・実行タブの左右ペインは初期表示で 1:1 を基本とする。
- 右ペインの結果テーブルが過大な初期要求幅を持たないように調整する。
  - 初期列幅の見直し
  - 必要に応じた列伸縮設定
- ウィンドウリサイズ時も極端に片側へ寄らないこと。

## 4. UI要件（詳細）
- 統合結果テーブルの最低列:
  - `対象ID`
  - `窓`
  - `判定/結果`
  - `理由`
  - `出力先`
- `対象ID` は `source:station_key:graph_type:base_or_annual` 形式とする。
- `窓` はイベント系で `3day` / `5day`、年最大系は空。

## 5. バリデーション要件
- 実行前検証開始前に以下をチェック:
  - 観測所1件以上
  - グラフ種別1件以上
  - イベント系を選んだ場合は基準日1件以上
  - イベント系を選んだ場合は `event_window_days_list` が空でないこと
- バッチ実行開始前に以下をチェック:
  - 実行前検証で `ready` になった対象が1件以上

## 6. 受け入れ条件
1. イベント窓を `3日` と `5日` の両方チェックして実行すると、イベント系は2窓分の対象が生成される。
2. イベント窓未選択時、実行前検証・バッチ実行ともにエラー表示される。
3. 年最大系のみ選択時、イベント窓未選択でも実行可能である。
4. 基準日はプルダウン選択 + 追加/削除で管理でき、自由記述は存在しない。
5. 基準日CSVをエクスポートし、同ファイルを再インポートして同一リストを再現できる。
6. 実行前検証と実行結果が同一テーブル上で追跡できる。
7. イベント系出力パスに `3day/5day` が含まれる。
8. 観測所選択変更で、基準日候補が選択観測所に連動して更新される。
9. チェック式の観測所選択と `全選択/全解除` が利用できる。
10. 観測所選択リストで `source` が `気象庁` / `水文水質DB` 表示になっている。
11. 条件設定・実行タブ初期表示で左右ペインが概ね1:1で表示される。

## 7. 完了条件（Definition of Done）
1. `tabs_execute` のUIが本書どおり更新されている。
2. DTOが `event_window_days_list` に統一され、旧単一値依存が残っていない。
3. `precheck` と `batch` のターゲット展開が窓ごとに分岐している。
4. 統合結果テーブルで `precheck_ng/ready/success/failed/skipped` が表示できる。
5. 単体テスト・結合テストが追加され、主要シナリオが通る。
6. `current` 文書（`spec.md` ほか）が実装内容に更新される。
7. R-EXE-08〜09 反映時、候補算出ルール（和集合）とUI操作仕様が `current/spec.md` に更新される。
8. R-EXE-10〜11 反映時、観測所表示形式とペイン比率仕様が `current/spec.md` に更新される。

## 8. 実装タスク分解（実装直結）
1. UI: 基準日 `Text` を `Combobox + Listbox + Add/Remove` へ置換。
2. UI: 基準日CSV `Import/Export` ボタン追加。
3. UI: イベント窓 `Radiobutton` を `Checkbutton` へ置換。
4. UI: 結果表示を単一Treeviewへ統合。
5. ViewModel: `event_window_days_list` と基準日リスト管理を実装。
6. DTO: `PrecheckInput/BatchTarget` の窓指定をリスト前提へ変更。
7. Service: イベント系ターゲットを窓数分展開。
8. Service: 出力パスへ窓ディレクトリ追加（イベント系のみ）。
9. Test: 単体/結合テスト追加。
10. Doc: `current` 更新。
11. UI: チェック式の観測所選択と `全選択/全解除` の追加。
12. ViewModel: 観測所連動の基準日候補再計算。
13. UI/ViewModel: 観測所 `source` 表示の日本語マッピング実装。
14. UI: 左右ペイン 1:1 設定、結果テーブル列幅/伸縮の初期調整。

## 9. テスト観点（最小）
- 正常:
  - 3dayのみ
  - 5dayのみ
  - 3day+5day
- 異常:
  - イベント系選択 + 窓未選択
  - 不正CSV（列不足、日付不正）
- 回帰:
  - 年最大系のみ実行
  - 停止処理
  - 既存ログ表示
  - 観測所大量件数での操作遅延
