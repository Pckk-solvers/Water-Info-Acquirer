# 条件設定・実行タブ: デフォルトスタイル実行前確認 要件定義

Status: target
Updated: 2026-04-01

## 1. 目的
- バッチ実行時にスタイルが初期値のままでも気づかず大量出力してしまう操作ミスを防ぐ。
- 実行前に「スタイル調整へ戻る」か「そのまま実行する」かをユーザーが選択できるようにする。

## 2. スコープ
- 対象:
  - `バッチ実行` 押下時の事前確認フロー
  - デフォルトスタイル判定ロジック
  - 確認ダイアログの表示抑止設定（`次回から表示しない`）
- 非対象:
  - スタイル編集UIそのものの機能追加
  - 描画アルゴリズムや出力フォーマット変更

## 3. 機能要件

### R-DSC-01 デフォルトスタイル判定
- `バッチ実行` 開始直前に、実行対象のスタイルが「実質デフォルト」かを判定する。
- 判定は `default_style()` と現在スタイルを同一ルールで正規化したうえで比較する。
- 判定対象は実行時に使う最新スタイル（UI編集中の内容を含む）とする。

### R-DSC-02 実行前確認ダイアログ
- R-DSC-01 が真の場合のみ確認ダイアログを表示する。
- メッセージは「スタイルが初期値のままです。スタイル調整しますか？」を基本とする。
- ボタンは以下の3択とする。
  - `スタイル調整へ移動`
  - `そのまま実行`
  - `キャンセル`

### R-DSC-03 ボタン動作
- `スタイル調整へ移動`:
  - バッチ実行を開始しない。
  - スタイル調整タブへ移動する。
- `そのまま実行`:
  - ダイアログを閉じて、既存フローのまま実行を継続する。
- `キャンセル`:
  - ダイアログを閉じて、実行を中断する（状態を変更しない）。

### R-DSC-04 表示抑止設定
- ダイアログに `次回から表示しない` チェックを持たせる。
- チェック有効時は、以降デフォルトスタイル判定が真でもダイアログを表示しない。
- 表示抑止設定はアプリ再起動後も保持する。

## 4. 受け入れ条件
1. スタイル未変更（デフォルト）のときだけダイアログが表示される。
2. スタイルを1項目でも変更した場合はダイアログが表示されない。
3. `スタイル調整へ移動` で実行せずにスタイルタブへ遷移する。
4. `そのまま実行` で従来どおりバッチ実行が開始する。
5. `キャンセル` で実行が開始しない。
6. `次回から表示しない` を有効にすると再起動後も表示されない。

## 5. テスト観点（最小）
- 正常:
  - デフォルト時の3ボタン分岐
  - 非デフォルト時に未表示
- 回帰:
  - 既存の実行前検証NG/READY判定を壊さない
  - 既存の停止処理、ログ表示を壊さない
