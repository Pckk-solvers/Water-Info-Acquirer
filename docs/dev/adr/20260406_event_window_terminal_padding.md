# ADR: イベント系窓の終端に1時間の余白を持たせる

## 状態
提案

## 背景
- Hydrology Graphs のイベント系は、基準日を中心とした 3日 / 5日 窓でデータを切り出して描画している。
- `water_info` / `jma` の hourly データは datetime 正規化済みで、24時相当は翌日 `00:00:00` として保持される。
- 現行の厳密な終端切り出しでは、その 24時相当を窓外として取りこぼす可能性がある。

## 決定
- イベント系は logical window（基準日中心の 3日 / 5日）とは別に capture window を持つ。
- capture window は logical window の終端側に 1時間の余白を追加する。
- この余白の有効・無効は描画設定から選択できる前提で扱う。

## 理由
- 24時相当の値を欠損扱いにせず、描画と検証の整合を保つため。
- logical window と capture window を分けることで、基準日を中心にした仕様を保ったまま境界だけを調整できるため。

## 影響
- 窓内の確認対象が1時間増える。
- 検証、プレビュー、バッチ実行で同じ窓定義を共有する必要がある。
- 境界条件のテストが追加で必要になる。
- 3日/5日 同時選択時の再利用は capture window 前提で行い、logical window のみを再切り出ししない。
