# 雨量 Parquet 期間抽出 要件

## 1. 目的

Parquet に保存済みの雨量データから、指定観測所・指定日を対象に CSV を出力する。

本機能は「取得済み Parquet の再利用」を目的とし、外部サイトへの追加リクエストは行わない。

## 2. 対象スコープ

### 対象

- Parquet からの読込
- GUI 新規タブ
- CSV 出力

### 非対象

- 外部データ再取得
- Excel / グラフ生成
- 共通出力ディレクトリとの統合

## 3. 機能要件

### R-01 Parquet 入力

- ユーザーは Parquet を含む出力ディレクトリを指定できること
- 想定入力は `parquet/` サブディレクトリを含む既存出力ディレクトリであること

### R-02 観測所選択

- ユーザーは観測所を外部マスタから検索するのではなく、指定した Parquet ディレクトリのスキャン結果から選択できること
- 選択候補は少なくとも以下を識別できること
  - `source`
  - `station_key`
  - `station_name`
- GUI 上の観測所入力は自由入力ではなく、Parquet 一覧ベースの選択 UI とすること

### R-03 期間指定

- ユーザーは `日付` を指定できること
- 第1段階では「単日指定」を正式要件とする
- 入力形式は `YYYY-MM-DD` とし、内部的にはその日付の 00:00〜23:00 の 24 時間を対象とする
- `年` は独立入力にしないこと

### R-04 CSV 出力

- 出力列は少なくとも以下を含むこと
  - `date`
  - `hour`
  - `rainfall`
- `date` は `YYYY-MM-DD`
- `hour` は既存 Excel と同じ 1-24 表記
- `rainfall` は 1時間雨量値

### R-05 時刻正規化

- CSV 出力の時間軸は、既存 Excel と同じ正規化ルールを使うこと
- JMA / WaterInfo の差異は Parquet 保存済み `observed_at` と `build_hourly_timeseries_dataframe()` の結果を基準に扱うこと

### R-06 欠測の扱い

- 指定日の 24 時間を必ず出力すること
- 値が存在しない時間は `rainfall` を空欄にすること

### R-07 出力ファイル名

- 初期要件としてファイル名は `{source}_{station_code}_{station_name}_{date}.csv` とする
- 理由:
  - 同一観測所で別日の出力時に衝突を避けるため
  - WaterInfo と JMA の衝突を避けるため source を入れる

### R-08 出力先

- 本機能は既存タブの共通出力先を使わないこと
- 新タブ専用の出力ディレクトリを個別指定できること

### R-09 不完全年の扱い

- 本機能では complete / incomplete 判定を出力可否条件にしないこと
- 指定日のデータを読めるなら出力対象に含めること

## 4. GUI 要件

### G-01 新規タブ

- 雨量 GUI に新規タブを追加すること
- 仮称: `期間CSV出力`

### G-02 入力項目

- Parquet 入力ディレクトリ
- 出力ディレクトリ
- Parquet スキャン結果からの観測所選択
- 対象日付

### G-03 実行結果

- 出力 CSV パスをログに表示すること
- 入力不備・Parquet 未検出・対象日データなしはユーザーに明示すること

## 5. 現行実装との矛盾点・注意点

### C-01 generate との責務衝突

- 既存 `run_rainfall_generate()` は「完全年の Excel / グラフ生成」が責務
- 今回は「単日 CSV 抽出」なので、同一 service に混ぜると責務が崩れる
- 新規 service として独立させるべき

### C-02 GUI 共通出力との衝突

- 既存整理・出力タブは共通出力ディレクトリ前提
- 今回は独立出力ディレクトリが必要なので、同タブへの追加は UX 上不自然

### C-03 JMA / WaterInfo の Parquet 粒度差

- JMA は月単位
- WaterInfo は年単位
- 読込 API で source 差を吸収する必要がある

### C-04 station code 表記

- GUI 表示では JMA の内部キー `prefecture_block` を block_no 表示へ落としている箇所がある
- CSV ファイル名・ログ・選択値で「表示コード」と「内部 station_key」を混同しないこと

### C-05 観測所の選び方

- 既存取得タブは都道府県・コード入力から観測所を解決する UI だが、本機能はその前提と異なる
- 今回は「取得済み Parquet の再利用」が目的なので、観測所選択の基準は Parquet スキャン結果であるべき
- 既存取得タブの観測所 selector をそのまま流用すると、Parquet 未存在の観測所を選べてしまうため要件と矛盾する

## 7. 推奨方針

第1段階は以下が最も安全。

- source は JMA / WaterInfo 両対応
- 期間指定は単日 `YYYY-MM-DD`
- 出力は 24 行固定
- CSV 列は `date,hour,rainfall`
- 観測所は Parquet スキャン結果から選択
- 出力ファイル名は日付を含める
- 新規 service + 新規 GUI タブで実装
