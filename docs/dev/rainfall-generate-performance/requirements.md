# 整理・出力タブ 性能改善 要件（Excel / グラフ）

## 1. 目的
- 「整理・出力」タブの処理時間を短縮し、長年数データでも実用速度を確保する。
- 出力内容（Excel列構成、グラフ仕様、Parquet入力仕様）は互換維持する。

## 1-1. 実施方針（決定）
- フェーズ1は **Excel高速化を必須** とする。
- グラフ高速化はフェーズ2で段階導入とする（スキップ + 差分生成のみ）。
- Excelもフェーズ2で差分更新を導入する。
- 差分モードの既定値は **ON** とする。
- `全再生成` 指定時は差分判定より優先し、必ずフル再生成する。
- GUI露出は最小構成（`差分更新を使う` / `全再生成する` の2項目）とする。

## 2. 対象
- `src/river_meta/services/rainfall.py` の `run_rainfall_generate()`
- `src/river_meta/rainfall/excel_exporter.py`
- `src/river_meta/rainfall/chart_exporter.py`
- 必要に応じてGUI設定（整理・出力タブ）

## 3. 機能要件
### R-01 観測所単位バッチ処理
- 現行の「観測所×年」ループを「観測所単位」処理へ再構成する。
- 同一観測所の全対象年をまとめて読み込み・集計し、Excel出力は1回で完了すること。

### R-02 Excel出力の重複I/O削減
- 同一ファイルへの年ごと再オープンを廃止する。
- 列幅自動調整はファイル作成時に1回のみ実行する。

### R-03 グラフ再生成スキップ
- 既存PNGがあり、入力データ更新がない場合は生成をスキップできること。
- スキップ時はログに件数を出すこと。

### R-04 差分生成モード（グラフ）
- 再実行時に「未生成の年/指標のみ」出力するモードを提供する。
- 既定挙動は互換維持（必要なら全再生成）。

### R-05 差分更新モード（Excel）
- 既存Excelがある場合、未出力年または更新が必要な年のみを追記/再生成できること。
- 既定挙動は互換維持（必要なら全再生成）。
- 差分判定根拠（例: Parquet更新時刻、処理メタ情報）をログに残すこと。
- 既定値はON。
- `全再生成` が有効な場合は差分判定を行わず、対象観測所をフル再生成すること。

### R-06 既存仕様維持
- 不完全年のスキップ仕様を維持する。
- 停止要求とエラー処理仕様を維持する。
- 出力ファイル名規約を維持する（互換性優先）。

### R-07 ログ可視化
- 以下をログ出力:
  - 観測所ごとの処理時間
  - Excel処理時間 / グラフ処理時間
  - 生成件数とスキップ件数（Excel/グラフ別）
  - 全体処理時間

## 4. 非機能要件
### NFR-01 性能目標
- 同一入力で現行比:
  - Excelのみ: 40%以上短縮（3年以上の同一観測所ケース）
  - グラフ再実行: 既存PNGスキップ有効時に70%以上短縮
  - Excel再実行: 差分更新有効時に70%以上短縮（未変更年が大半のケース）

### NFR-02 安定性
- 既存の回帰テストが通ること。
- 途中停止・途中エラー時にも結果整合性を保つこと。

### NFR-03 保守性
- 高速化のON/OFFを設定で制御可能にする。
- 追加テストで通常モード/高速モードの両方を検証する。

## 5. 受け入れ条件
1. 同一観測所3年ケースで、Excelのみ処理時間が現行より短縮される。
2. 同一観測所3年ケースで、Excel出力時のファイルオープン回数が観測所単位に削減される。
3. グラフ再実行で既存PNGをスキップできる。
4. グラフ差分生成で未生成の年/指標のみ作成できる。
5. Excel差分更新で未出力年/変更年のみ更新できる。
6. 不完全年スキップ、停止要求、エラー処理の既存挙動を維持できる。
7. 自動テスト（単体/結合）が追加され、回帰が通る。
8. 既定起動時に差分モードONで動作する。
9. `全再生成` 指定時に差分判定をバイパスしてフル再生成される。
