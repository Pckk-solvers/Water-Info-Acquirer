# 整理・出力タブ 性能改善 Overview（Excel / グラフ）

## 1. 対象
- GUI「整理・出力」タブで実行される `run_rainfall_generate()` の性能。
- 具体的には、Parquetからの読み込み、時系列/年最大集計、Excel書き出し、グラフPNG出力。

## 2. 現行実装フロー（事実）
- 入口: `src/river_meta/rainfall/gui.py` -> `run_rainfall_generate()`
- 実装: `src/river_meta/services/rainfall.py:121`
  - `scan_parquet_dir()` で観測所×年エントリ作成
  - 完全エントリを年単位で順次処理
  - 年ごとに:
    - Parquet読込（JMAは月12ファイル結合）
    - `build_hourly_timeseries_dataframe()`
    - `build_annual_max_dataframe()`
    - Excel出力（有効時）
    - グラフ出力（有効時）

## 3. ボトルネック（原因）
1. 年単位処理のため、同一観測所で同じ計算・I/Oを毎年繰り返す。
2. Excel出力は同一観測所ファイルに対して年ごとに `openpyxl` で再オープン/追記しており、年数増加で非線形に遅くなる。
3. `excel_exporter.py` の列幅自動調整が毎回シート全体走査になるため、追記回数が増えるほど重くなる。
4. グラフは「年 × 指標6枚」を毎回再生成するため、年数にほぼ線形で時間が増える。
5. 既存PNGの再利用判定がなく、再実行時にも全画像を作り直す。
6. Excelも再実行時の差分判定がなく、既存ファイルがあっても同様の重い処理を繰り返す。

## 4. 簡易実測（2026-03-05、ローカル）
- 条件: 1観測所（JMA）、1年=12ヶ月Parquet、8760時間データ
- 実測結果:
  - Excelのみ:
    - 1年 `9.36s`
    - 2年 `20.74s`
    - 3年 `44.97s`（非線形）
  - グラフのみ:
    - 1年 `12.98s`（6枚）
    - 2年 `28.19s`（12枚）
    - 3年 `43.05s`（18枚）
  - 両方（2年）: `56.24s`

## 5. 改善方針（優先度）
### P0（最優先）
1. 同一観測所を「年ごと」ではなく「観測所まとめて」処理する。
   - 読み込み・集計を観測所単位で1回にする。
   - Excelも観測所ごとに1回だけ書き出す。
2. Excel列幅自動調整の実行回数を1回に制限する（追記ごとに全走査しない）。

### P1（高優先）
3. グラフ生成のスキップ条件を導入（既存ファイル再利用）。
4. グラフの差分生成（未生成の年/指標のみ生成）を導入。

### P2（中優先）
5. Excelの差分更新（未出力年のみ追記、または変更年のみ再生成）を導入。
6. 出力指標の選択機能（全6指標固定から選択可能へ）を導入。

## 6. 期待効果
- 同一観測所・複数年でのExcel処理時間を大幅短縮（非線形劣化の抑制）。
- 再実行時の無駄なPNG再生成を削減。
- 大量年数・複数観測所の運用で体感速度を大きく改善。

## 7. 方針決定（2026-03-05）
- 実装は **Excel先行** とする。
  - まず P0 の観測所単位バッチ化 + 列幅自動調整回数削減を実装する。
- グラフは第2フェーズで改善する（並列化は今回は実施しない）。
  - 「既存PNGスキップ」と「未生成分のみ差分生成」を導入する。
- Excelも第2フェーズで差分更新を導入する。
  - 既存Excelとの差分（未出力年/変更年）を判定して再出力範囲を最小化する。
