# Hydrology Graphs architecture

`src/hydrology_graphs/` は、雨量・水位グラフの作成と条件設定、プレビュー、スタイル管理を担当する。

## 入口

- `src/hydrology_graphs/launcher_entry.py`
  - アプリ選択ランチャーから開く入口。
- `src/hydrology_graphs/__init__.py`
  - パッケージの入口。

## 責務

- グラフ作成条件の入力
- イベント系の基準日窓に対する終端余白は常時有効
- スタイル調整タブでの表示モード切替（1時~24時 / datetime）
- プレビュー出力対象の観測所・基準日・対象グラフを、`precheck OK` の実在候補に絞って選ばせる
- スタイル調整
- プレビュー表示
- バッチ実行
- parquet 入力のスキャンと選択
- しきい値・スタイルの保存と読込

## 主な層

- `ui/`
  - 実行タブ、スタイルタブ、プレビュー、イベント処理、ビュー模型を置く。
  - スタイル調整タブ左側の表示モード設定を `style.display.time_display_mode` として扱い、JSON と同期する。
  - `プレビュー出力対象` では、観測所 / 基準日 / 対象グラフを並べて扱い、`プレビュー更新` ボタンは右端に置く。
  - プレビュー対象の 2 行目の説明ラベルは廃止し、候補が不整合なら `precheck OK` の実在候補へ補正する。
  - プレビュー候補の解決は `観測所 / 基準日 / 対象グラフ` の完全一致に限定し、合わない場合は自動フォールバックしない。
  - 条件設定・実行タブの観測所一覧は、軽量スキャンでファイル名から推定したメトリクス候補を併記する。気象庁は雨量固定として扱う。
  - イベント窓補正は GUI から外し、常時有効の条件として `services/` へ渡す。
  - 表示モードに応じて、preview/batch の対象窓を `datetime` 用と `24時表記` 用で切り替える。
  - `24時表記` の X 軸は時だけを表示し、`24` が前日終端として自然に見えるようにする。
- `services/`
  - UI からの要求をビジネスルールへ変換する。
- `domain/`
  - グラフ種別、時刻窓、集計ルール、定数、モデルを置く。
- `render/`
  - 実際の描画処理を置く。
- `io/`
  - parquet、style、threshold、PNG 出力の I/O を置く。
  - 観測所一覧向け軽量スキャンでは、ファイル名由来のメトリクス候補を補う。
- `ui/`
  - 結果一覧の日本語ラベル整形と内部ID保持を分けて扱う。

## データの流れ

1. UI が対象データ、期間、グラフ種別、スタイルを受け取る。
2. スタイル調整タブで表示モード（24時表記 / datetime表記）を選ぶ。
  3. `プレビュー出力対象` で観測所 / 基準日 / 対象グラフを選ぶ。候補は `precheck OK` の実在組み合わせに絞られ、入力不整合は補正する。
  4. プレビュー解決時は `観測所 / 基準日 / 対象グラフ` の完全一致だけを採用し、別候補へのフォールバックはしない。
  5. UI が表示設定を含めて条件を組み立てる。イベント系の終端余白は常時有効として扱い、表示モードに応じて対象窓も切り替える。
  6. `services/usecases.py` がスキャン・検証・プレビュー・バッチの順を制御する。
  7. `domain/logic.py` が logical window（3日/5日）と、24時相当を保持するための capture window を決める。入力 parquet は `water_info` / `jma` の datetime 正規化済みデータを前提とする。
  8. `render/plotter.py` が capture window のデータをもとに、選択された表示モードに応じた時刻表記でグラフを描画する。
  9. `io/` が parquet 読込、スタイル保存、PNG 保存を担当する。
  10. `ui/` が結果一覧の `target_id` を内部管理しつつ、日本語の表示ラベルへ変換する。

## 補足

- この package は「条件設定・実行タブ」の本体。
- `style` と `threshold` は保存形式が別で、UI から更新する。
- 表示モードはスタイル JSON の一部として保存し、GUI の専用設定から変更する。
- `datetime` と `1時~24時` では、X軸ラベルだけでなく切り出すデータ範囲も変える。
  - この切り出し切替は precheck / preview / batch で同じ判定を使う。
- `1時~24時` の X 軸表示は時だけを使い、分は出さない。
  - 観測所一覧のチェック行サンプルは `観測所A (気象庁:001) / 雨量`、`観測所B (水文水質DB:002) / 流量`、`観測所C (水文水質DB:003) / 水位` のように表示する。
- グラフ作成前のスキャンや確認処理もここに含まれる。
- イベント系は、基準日を中心とした論理窓と、24時相当を落とさないための終端余白付き窓を分けて扱う。
- イベント窓補正は常時有効で、GUI で切り替えない。
- 24時相当は source 側で翌日 `00:00:00` に正規化済みだが、描画前の窓端判定では終端余白が必要になる。
- 3日/5日の再利用は capture window 前提で行い、logical window の切り出しだけに戻さない。
- 結果一覧の `対象` 列は内部IDではなく `高幡橋（気象庁:111） / ハイドログラフ（水位） / 2025-01-02 / 3日窓` のような日本語ラベルで表示する。
