# Water Info 時刻正規化・出力整合 要件（現行）

## 1. 背景
- `water_info` の時間データは source 上で `1:00..24:00` の表記を持つ。
- 現行では値列を順番に拾ったあと、後段で時刻を付与する経路があるため、時刻列とデータ列の対応がずれる余地がある。
- Excel 出力では `period_end_at` を正として扱うため、source の見た目と出力の見た目が一致しない場合がある。

## 2. 目的
- 時刻と値を取得時点で 1 レコードとして確定し、時刻列とデータ列のずれを防ぐ。
- source 側でも最初から `datetime` / `period_end_at` を持つ形に寄せ、後段の出力と整合させる。
- 24時相当は翌日 `00:00:00` として正規化し、時刻表現のルールを統一する。

## 3. スコープ
- 対象: `src/water_info`
- 対象処理:
  - HTML 取得結果のパース
  - 時刻と値のペアリング
  - unified record の生成
  - Excel / Parquet 出力
- 対象外:
  - `jma_rainfall_pipeline` 側の入力契約
  - `hydrology_graphs` 側の描画ロジック

## 4. 入力
- 水文水質データベースの時間データ HTML
- 水文水質データベースの日データ HTML
- HTML 内の時刻セル、値セル、日付セル

## 5. 出力
- `datetime` / `period_end_at` / `observed_at` を持つ unified record
- 時刻と値が対応した Excel 出力
- 時刻正規化済みの Parquet 出力

## 6. 制約
- 時刻は `datetime` を正とし、必要に応じて `period_end_at` を導出する。
- 24時相当は翌日 `00:00:00` として保持する。
- 値列は時刻と同じ行に対応づくことを前提とし、単なる配列の順番だけに依存しない。
- 表示用の別時刻列を恒久保存しない。

## 7. 完了条件
- 取得時点で時刻と値が対応したレコードが生成される。
- source 側の中間データでも `datetime` を確認できる。
- Excel 出力で時刻と値のずれが発生しない。
- 24時相当の表示・保存ルールが一貫している。
